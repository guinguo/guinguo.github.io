一，课内学习

首先是汇编，要求是把一个16位的二进制数按，eg: BX:0110101100111100

  • 二进制
  • 四进展
  • 八进制
  • 十进制
  • 十六进制
    分别显示出来。
    二进制的：使用ROL循环左移命令，循环移动16次输出
    然后剩下的我的做法是：
    四进制，循环八次，初始值为30H，每次第一位如果是1，我就ADD 2，第二位如果是1，我就ADD 1，最后输出，0的话就什么都不加。
    这样结果就是，1230330
    八进制要想输出第一位，然后循环5次移动3位，做法类似四进制，十六进制同理。

不过
老师的做法更简单，直接移位
四进制：直接移动2位，

ROL BX,CL
MOV DL,BL
AND DL,3  #保留低2位

…循环输出

八进制，同样要输出第一位，然后每次移动3位，十六进制就是4位。这样的代码量比较少，也更灵活。

十进制有两种做法：

  1. 循环5次，先除以10000，输出商，余数继续除以1000，100，10，这样就可以得出结果
  2. 也是循环五次，每次除以10，余数入栈，商继续除。

然后是计组，快速过了一门硬件语言VHDL的语法，老师讲得有点快，基本没记住。。。
VHDL语言是一种用于电路设计的高级语言。它在80年代的后期出现。最初是由美国国防部开发出来供美军用来提高设计的可靠性和缩减开发周期的一种使用范围较小的设计语言 。
实验课写了个半加器，

library ieee; //引入库
use ieee.std_logic_1164.all;//有点像c++

entity fa is //这是这门语言的数据结果，类似一个实体类
port(a,b,ci : in std_logic; //io变量的定义
       s,co : out std_logic);
end fa;

architecture b_fa of fa is //一个使用fa的结构体
begin
  s<=a xor b xor ci;// s=a异或b异或ci
  co<=((a xor b) and ci) or (a and b);//and 是与运算
end b_fa;

有点像汇编的写法。。。

操作系统呢，主要教了互斥与同步。在单处理器多道程序设计系统中，进程被交替执行，表现出一种并发执行的外部特征。这时，有些资源的访问或共享充满了危险，比如两个进程都使用同一个变量，进行读写操作，那么结果不可预料。

讲到互斥，就要提到临界区，所谓临界区，可以是一段代码，或者是某些资源，比如说打印机，在一个时刻下，只能有一份文件在打印。其他人要打印就必须排队等待，这里的排队等待就使用了互斥原理，当打印完成后，打印机发出信号，我空闲了，那么下一个人马上去打印，这里就体现了同步。

常用的解决并发问题有一个就是信号量，什么是信号量呢？就那上面的打印栗子讲，这里的信号量可以是打印机是否被使用。有三个步骤。

  1. 一个二元信号量的初始化，0或者1.
  2. semWait操作检查信号量的值，若为0，那么semWait就会受阻。若为1则改变为0，继续执行改进程。
  3. semSignal操作检查是否有任何进程在该信号上受阻，有的话，受阻进程就会被唤醒，如果没有进程受阻，那么值就被设为1.
while(true){
    semWait(s);
    /*临界区*/
    semSignal(s);
    /*ohter code*/
}
    使用信号量的互斥

还有一个经典的问题，就是生产者跟消费者的问题，我就不描述，直接给出代码了

void producer(){
     while(true){
     produce();
     semWait(s);//等待进入缓冲区
     append(); //向缓冲区写入
     semSignal(s);//释放缓冲区
     semSignal(n); //缓冲区数据+1；
     }
}

void consumer(){
     while(true){
     semWait(n);//判断缓冲区释放有数据可读
     semWait(s);//等待进入缓冲区
     take(); //从缓冲区读出
     semSignal(s);//释放缓冲区
     consume();
     }
}
