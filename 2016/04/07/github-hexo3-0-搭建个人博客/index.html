发表于 2016-04-07   |   分类于 blog   |  

一、前言

最新好奇如何搭建个人博客网站，由于之前买过一台vps，本打算自己写页面的，但是想想还是算了吧，毕竟hexo已经有很多酷炫的主题了，之后有空再自己撸。。。

由于此类教程在网上已经有很多人发布了。我就根据我在搭建环境，配置参数的情况，做一点小总结：

三、遇到的问题

1,

这些的教程写得非常详细，我是参考他们的教程搭的，然后我在搭的时候，安装hexo老是出问题，是npm install 不了，卸载了重装也是一样，最后发现了一个很无语的原因–npm install 没反应

2,

完成后就可以托管在github上面了，最后可以绑定自己的域名，这里我也遇到了一个坑，因为修改dns后散播需要一段时间，才会有效，1到几个小时不等吧。然后发现我绑定的域名老是404.那些教程都说是

在repo的根目录下建立一个CNAME的文件里面写上你的顶级域名，不用加www.

而我发现github上面还是没有CNAME这个文件，原来这个文件是要放在项目根路径的source文件夹里面。这样才不会被hexo清理掉导致没有推到github。

我绑定的是二级域名www.guinguo.top 所以需要加上www.

最后只需要在域名后台管理的dns上加一条CNAME记录

前面填www，然后指向guinguo.github.io（改成你自己的）就行

最后就可以尽情的撸文章了
