dtd基础入门

发表于 2016-04-24   |   分类于 WebService   |  

一、What is DTD

DTD (Document type definition)文档类型定义, wiki
用于规定xml或者html之类的标记性文档的类型

二、How to use it

直接举一个栗子，比如现在我定义了一个Person对象，

<person>
    <id></id>
    <name></name>
    <password></password>
    <age></age>
</person>

对应的dtd类型定义如下

<!DOCTYPE persons [
<!ELEMENT persons (person+)>
<!ELEMENT person (id,name,password,age)>
<!ELEMENT person id ID #REQUIRED>
<!ELEMENT id (#PCDATA)>
<!ELEMENT name (#PCDATA)>
<!ELEMENT password (#PCDATA)>
<!ELEMENT age (#PCDATA)>

其中persons 是根元素，每个元素都要说明，persons中有person就如第二行，+号表示1次或者多次，跟正则语法一样

用修饰符，(正则)，?表示0次或者1次，*表示0次或者多次，+号表示1次或者动词，默认表示1次
其中person中，逗号的属性是必须按照顺序写的，即引用了次dtd的xml的person节点的name节点不能出现在id节点前

可以把id设置为属性
声明属性,使用下列语法：

<!ATTLIST 元素名称 属性名称 属性类型 默认值>
<!ATTLIST person id ID #REQUIRED>
其中类型有

类型                 描述 
CDATA        值为字符数据 (character data) 
(en1|en2|..) 此值是枚举列表中的一个值 
ID           值为唯一的 id 
IDREF        值为另外一个元素的 id 
IDREFS       值为其他 id 的列表 
NMTOKEN      值为合法的 XML 名称 
NMTOKENS     值为合法的 XML 名称的列表 
ENTITY       值是一个实体 
ENTITIES     值是一个实体列表 
NOTATION     此值是符号的名称 
xml:         值是一个预定义的 XML 值

默认值参数可使用下列值：

值              解释
值              属性的默认值
#REQUIRED       属性值是必需的
#IMPLIED        属性不是必需的
#FIXED value    属性值是固定的

如果把sex设置为属性，就可以写：默认为男性

<!ATTLIST person sex PCDATA 男>

但是，为person增加id属性，为必须的,id不能以数字开头…

三、Why Use a DTD?


使用DTD，每个XML文件可以携带自己的格式的描述。
使用DTD，团体中的每个人可以使用标准的DTD交换数据。
应用程序可以使用标准的DTD来验证从外界接收到的数据是有效的。
另外您可以使用DTD，以验证自己的数据。

四、XML文件引入

<!DOCTYPE SYSTEM 是引用内部文件的
<!DOCTYPE PUBLIC 是引用网络文件的

例如：引入内部文件person.dtd
<!DOCTYPE persons SYSTEM “person.dtd”>

Practice

下面给出两道练习题：
1,

<persons>
 <person id="p1">
  <sex></sex>
  <name></name>
  <age></age>
  <messages>
   <message>
    <title></title>
    <content></content>
   </message>
  </messages>
 </person>
</persons>

2,

<classroom id="c1">
 <claName>网工3班</claName>
 <grade>2014</grade>
 <students>
  <student>
   <id>s1</id>
   <stuName>guin</stuName>
   <age>21</age>
  </student>
  <student>
   <id>s1</id>
   <stuName>guin</stuName>
   <age>21</age>
  </student>
 </students>
</classroom>

但是，使用dtd会出现一些问题，
1，名称会冲突，如果person有name，message也有name就有问题
2，控制精度不够，比如sex应该为枚举类型，只有男跟女
导致dtd并不能适应很多场合，跟xml进行互换
在09年出现了schema，慢慢在xml的定义中替代了dtd
