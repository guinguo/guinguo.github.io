Hadoop-Cluster-Set-Up-P1_Prepare

发表于 2016-07-23   |   分类于 BigData   |  

前几天买了4台vps来搭建集群，具体情况如下：
本来要进行分区，存储数据的，但是这个类型的vps居然不支持重新分区。。。
Centos7

Server1 master(NameNode)
Server2 master(SecondaryNameNode)
Server3 slave1(DataNode)
Server4 slave2(DataNode)

改hosts文件，增加Hadoop用户

拿到vps后，修改hosts文件

vi /etc/hosts

像这样一行一行把ip地址映射上去
192.168.1.1 Server1

useradd hadoop
echo "hadoop123"|passwd --stdin hadoop

授予sudo权限

echo 'hadoop  ALL=(ALL)       NOPASSWD: ALL' >> /etc/sudoers

配置ssh免密码登录

[root@server1 ~]# ssh-keygen -t rsa
三个回车，出现
The key’s randomart image is:

+--[ RSA 2048]----+
|E                |
|.                |
|o.               |
|o.     .         |
|o     . S .      |
|..     . +       |
|  +   . oo .     |
| . o o..+o=      |
|    .o=++=..     |
+-----------------+

将id-rsa.pub 公钥添加到.ssh/authorized_keys文件中

cat  id-rsa.pub >> authorized_keys

我使用复制方式，从server1复制到server2，然后添加server2的公钥，再复制到server3，4。
最后在server4就有了一份全部server的公钥文件，如果麻烦你也可以直接复制到一个文件。
最后分发到四台server。执行

chmod 600 .ssh/authorized_keys

好了，配置好了之后，注意:
第一次ssh登录时可能需要输入密码，再次访问时即可免密码登录。
还有我在每一台server是分别ssh其他三台server。因为出现：

The authenticity of host 'server1 (xx.xx.xx.xx)' can't be established.
ECDSA key fingerprint is c6:03:ee:f5:60:0c:ae:76:d8:f3:ad:70:c5:66:04:2d.
Are you sure you want to continue connecting (yes/no)?

会把host记录在.ssh/known_hosts文件中，之后就可以直接ssh上，如果嫌麻烦可以
修改/etc/ssh/ssh_config
将其中的# StrictHostKeyChecking ask 改成 StrictHostKeyChecking no

安装jdk1.7

这里安装的是jdk-7u79-linux-x64，因为1.8会出现各种奇怪的问题，之前在虚拟机就遇到过，所以还是装1.7的好
下载jdk-7u79-linux-x64.tar.gz，并上传到server上，
如果是国外的服务器，ftp上传慢的话，课使用

wget --no-cookies --no-check-certificate --header "Cookie: gpw_e24=http%3A%2F%2Fwww.oracle.com%2F;oraclelicense=accept-securebackup-cookie" "http://download.oracle.com/otn-pub/java/jdk/7u79-b15/jdk-7u79-linux-x64.tar.gz"

这个命令，下载速度有7M/s

  • 解压安装

    tar -zxvf jdk-7u79-linux-x64.tar.gz -C /usr/local/java/

  • 配置环境变量

    vi /etc/profile
