Hadoop Cluster Set Up P6 Sqoop

发表于 2016-08-16   |   分类于 BigData   |  

Sqoop1.99.7 暂时只支持mysql到hdfs或者hdfs到mysql，不支持hive或者hbase。官网也说了不用与生产环境。这里我只是先装 (踩) 着 (坑)

下载并解压

由于服务器在美国，所以我换了一个镜像,大家如果在国内的话，可以使用北理工的http://mirror.bit.edu.cn

wget http://www-us.apache.org/dist/sqoop/1.99.7/sqoop-1.99.7-bin-hadoop200.tar.gz

tar -zxvf sqoop-1.99.7-bin-hadoop200.tar.gz 
mv sqoop-1.99.7-bin-hadoop200 sqoop-1.99.7

配置

  • 环境变量
/etc/profile

export SQOOP_HOME=/home/hadoop/sqoop-1.99.7
export PATH=$PATH:$SQOOP_HOME/bin
export CATALINE_BASE=$SQOOP_HOME/server
export LOGDIR=$SQOOP_HOME/logs

  • sqoop配置文件
    　　这里注意，配置文件不再存放于${SQOOP_HOME}/server/conf/目录下，而是直接存在安装的目录${SQOOP_HOME}/conf/下

    vi ${SQOOP_HOME}/conf/sqoop.properties

　　把下面的属性替换成hadoop的配置文件目录，注意！我第一次换成hadoop安装目录，然后就报错了，尴尬。。。

org.apache.sqoop.submission.engine.mapreduce.configuration.directory=/usr/local/hadoop-2.7.2/etc/hadoop/

　　并且替换@LOGDIR@ 和@BASEDIR@ ：

:0,$ s/@LOGDIR@/logs/g
:0,$ s/@BASEDIR@/base/g

vi ${SQOOP_HOME}/conf/sqoop_bootstrap.properties

　　这里要注意，有个坑，1.99.7版本之前，都是catalina.properties这个配置文件，然后我下了1.99.7就没有发现，而且官网也没显式说明，改名为sqoop_bootstrap.properties
把hadoop的所以jar都加进来

common.loader=${catalina.base}/lib,${catalina.base}/lib/.jar,${catalina.home}/lib,${catalina.home}/lib/.jar,${catalina.home}/../lib/.jar,/usr/local/hadoop-2.7.2/share/hadoop/common/.jar,/usr/local/hadoop-2.7.2/share/hadoop/common/lib/.jar,/usr/local/hadoop-2.7.2/share/hadoop/hdfs/.jar,/usr/local/hadoop-2.7.2/share/hadoop/hdfs/lib/.jar,/usr/local/hadoop-2.7.2/share/hadoop/mapreduce/.jar,/usr/local/hadoop-2.7.2/share/hadoop/mapreduce/lib/.jar,/usr/local/hadoop-2.7.2/share/hadoop/tools/.jar,/usr/local/hadoop-2.7.2/share/hadoop/tools/lib/.jar,/usr/local/hadoop-2.7.2/share/hadoop/yarn/.jar,/usr/local/hadoop-2.7.2/share/hadoop/yarn/lib/.jar,/usr/local/hadoop-2.7.2/share/hadoop/httpfs/tomcat/lib/.jar

  • 复制mysql-connector-java-*-bin.jar到${SQOOP_HOME}/server/lib/下

mysql-connector

cp mysql-connector-java-5.1.39-bin.jar ${SQOOP_HOME}/server/lib/

运行测试

  • 首先验证配置是否正确
sqoop2-tool verify

这里我又踩了一个坑

Verification has failed, please check Server logs for further details.
Tool class org.apache.sqoop.tools.tool.VerifyTool has failed.
查看日志 ${SQOOP_HOME}/logs/sqoop.log

MapReduce配置出错了？
搞了半天，无语啊，我配置了TEZ，mapreduce框架默认使用yarn作为管理，但是我在yarn-site.xml中配了tez

<property>  
  <name>mapreduce.framework.name</name>  
  <value>yarn-tez</value>  
</property>

把这个配置去掉，使用默认的yarn就正常了，，，sqoop2的兼容实在是不敢恭维。

[hadoop@server1 sqoop-1.99.7]$ sqoop2-tool verify
