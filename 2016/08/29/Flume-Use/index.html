Flume Use

发表于 2016-08-29   |   分类于 BigData   |  

　　Apache Flume是一个分布式，可靠性，可用性的系统，可以有效地从不同的数据源收集，整合，和移动大量日志数据到其他的存储（HDFS，HBase etc…）
flume其实没有集群概念,每个flume都是一个独立的个体，每个flume agent收集数据汇总到flume collector，由collector写入flume storage。

　　Flume event是一个有效字节数据单元和一组可选的字符串属性组成.Flume agent是一个JVM进程，控制组件事件流从外部流向目的地.Flume消费外部源传送给它的数据，像web server。外部数据源以Flume Source认可的形式传递给flume，例如，Avro Flume 源可以用来从Avro客户端或者从Flume代理的Avro 槽(sink) 流出的事件流接收Avro事件。类似的流可以使用第一Thrift Sink或者一个Flume Thrift Rpc 客户端，或者用任意语言编写的遵循Thrift协议栈的Thrift客户端。
当一个Sink 源接收到一个事件时，它可以存储到一个或者多个渠道(channel)。channel是存储事件直到事件被sink消费。文件渠道(channel)，就是一种，它支持本地文件系统，Sink 从channel移除一个事件并把它放到外部仓库像HDFS(通过Flume HDFS sink)，或者直接（转发）流向下一个数据源被其使用。指定agent中的source和sink 在channel上进行异步事件处理

以上翻译自flume官网，措辞不对请见谅！

使用

server1作为collector，server2 作为agent。实现agent连接collector然后想collector发送日志，最终collector将日志写到HDFS。

系统要求

  1. jdk 大于1.6，最好是1.7
  2. 内存 足够的内存配置以使用source，channels，sinks
  3. 磁盘空间 足够的磁盘空间使用source，channels，sinks
  4. 目录权限 agent可以直接读写响应目录

下载安装

[hadoop@server1 ~]$ wget http://www-us.apache.org/dist/flume/stable/apache-flume-1.6.0-bin.tar.gz
[hadoop@server1 ~]$ tar -zxvf apache-flume-1.6.0-bin.tar.gz 
[hadoop@server1 ~]$ mv apache-flume-1.6.0-bin flume-1.6.0

修改配置文件

jdk环境变量

[hadoop@server1 ~]$ mv flume-1.6.0/conf/flume-env.sh.template flume-1.6.0/conf/flume-env.sh          
[hadoop@server1 ~]$ vi flume-1.6.0/conf/flume-env.sh 
#把jdk目录填上去

复制到server2

[hadoop@server1 ~]$ scp -r flume-1.6.0/ hadoop@server2:~/

agent启动文件

在server1节点，conf目录下，将flume-conf.properties.template 重命名一份出来
用于启动配置。关于三大组件的详细配置，请参考官方文档

创建server2事件监听目录/home/hadoop/flume-test

  • server1上的配置

    [hadoop@server1 flume-1.6.0]$ cp conf/flume-conf.properties.template conf/avro.conf

    [hadoop@server1 flume-1.6.0]$ cat conf/avro.conf
    #defind source sink channel
    a1.sources = r1
    a1.sinks = k1
    a1.channels = c1

    #specify configure
    #r1的类型是Avro
    a1.sources.r1.type = avro
    #channel为c1，注意，这里是r1.channels
    a1.sources.r1.channels = c1
    #绑定本机端口4141
    a1.sources.r1.bind = 0.0.0.0
    a1.sources.r1.port = 4141

    #sink k1的类型为HDFS
    #see http://flume.apache.org/FlumeUserGuide.html#hdfs-sink
    a1.sinks.k1.type = hdfs
    a1.sinks.k1.hdfs.path = hdfs://server1:9000/flume-test/
    a1.sinks.k1.hdfs.fileType = DataStream
    a1.sinks.k1.hdfs.writeFormat = TEXT
    a1.sinks.k1.hdfs.round = true
    a1.sinks.k1.hdfs.roundValue = 5
    a1.sinks.k1.hdfs.roundUnit = minute
    a1.sinks.k1.hdfs.useLocalTimeStamp = true
    a1.sinks.k1.hdfs.filePrefix = events-
    #而这里是k1.channel
    a1.sinks.k1.channel = c1

    #使用内存作为channel
    a1.channels.c1.type = memory
    a1.channels.c1.capacity = 1000
    a1.channels.c1.transactionCapacity = 100

  • server2上的配置

    [hadoop@server2 flume-1.6.0]$ cat conf/avro.conf
    a1.sources = r1
    a1.channels = c1
    a1.sinks = k1

    #For each one of the sources, the type is defined
    a1.sources.r1.type = spooldir
    a1.sources.r1.spoolDir = /home/hadoop/flume-test
    a1.sources.r1.channels = c1

    #Each sink’s type must be defined
    a1.sinks.k1.type = avro
    a1.sinks.k1.hostname = server1
    a1.sinks.k1.port = 4141
    a1.sinks.k1.channel = c1

    #Each channel’s type is defined.
    a1.channels.c1.type = memory
    a1.channels.c1.capacity = 1000
    a1.channels.c1.transcationCapacity = 100

启动测试

　　
　　以上Avro配置是：server2监听目录/home/hadoop/flume-test，如果有文件变化，就把事件传递到server1的4141端口上，server1监听本机的4141端口，如果有事件接收，就传递到hdfs hdfs://server1:9000/flume-test/上存储下来 文件名前缀events-

  1. 先启动server1：

    cd bin
    bin/flume-ng agent -n agent -c conf -f conf/avro.conf -Dflume.root.logger=DEBUG,console

  2. 然后是server2：
    bin/flume-ng agent -n agent -c conf -f conf/avro.conf -Dflume.root.logger=DEBUG,console

  3. 进入目录
    [hadoop@server2 flume-test]$ cd /home/hadoop/flume-test
    [hadoop@server2 flume-test]$ echo “hello flume source is avro and write to hdfs” > test.txt

  4. 查看结果
