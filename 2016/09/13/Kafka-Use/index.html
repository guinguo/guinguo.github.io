Kafka Use

发表于 2016-09-13   |   分类于 BigData   |  

Kafka简介

　　Kafka是一个高吞吐量的分布式消息订阅发布系统，它采用独特的设计提供消息传递系统的功能。

术语

  • topic Kafka维护的消息流称为topic
物理上不同的topic的消息分开存储，逻辑上一个topic的消息虽然保存于一个或多个broker上，
但用户只需指定消息的topic即可生产或者消费数据而不必担心数据存于何处
  • producer 消息发布者称为producer
  • consumer 订阅并消费消息的称为consumer
  • broker Kafka集群中的每个节点称为broker
  • partition 物理概念，每个topic包含一个或者多个partition

下载解压

wget  http://archive.apache.org/dist/kafka/0.10.0.0/kafka_2.11-0.10.0.0.tgz
tar -zxvf kafka_2.11-0.10.0.0.tgz
mv kafka_2.11-0.10.0.0 kafka

配置分发

kafka自带zookeeper，但是我们使用我们自己的,找到zookeeper.connect 这一配置项，修改

cd kafka/config
vi server.properties
     zookeeper.connect=server1:2181,server2:2181,server3:2181

创建一个启动脚本用于启动Kafka

vi kafka/start.sh
nohup bin/kafka-server-start.sh     config/server.properties > kafka.log 2>&1 &
:wq
chmod +x kafka/start.sh

复制到其他节点

scp -r kafka server2:~/
scp -r kafka server3:~/

启动测试

  1. 首先启动zookeeper

    zkServer.sh start

  2. 再启动Kafka

    ./kafka/start.sh

  3. 使用jps命令查看

    [hadoop@server1 ~]$ jps
    19114 JournalNode
    19979 Jps
    18400 JobHistoryServer
    18479 ResourceManager
    18331 DFSZKFailoverController
    18277 QuorumPeerMain
    11797 NameNode
    12485 Kafka

    可以看到已经启动了

  4. 创建topic

    [hadoop@server3 ~]$ cd kafka
[hadoop@server3 ~]$ bin/kafka-topics.sh --zookeeper server1:2181 --create --topic test-kafka --replication-factor 2 --partitions 2
Created topic "test-kafka".

    这条命令指定了zookeeper节点为server1(随便指定一个都行，因为zk是一个集群) 创建一个名为test-kafka的topic，副本数为2

    可以查看zk上面是否已经有了这个topic

    [hadoop@server3 kafka]$ bin/kafka-topics.sh --zookeeper server1:2181 --list
test-kafka

  5. 查看topic描述

    [hadoop@server3 kafka]$ bin/kafka-topics.sh --zookeeper server1:2181 --describe --topic test-kafka
Topic:test-kafka    PartitionCount:2    ReplicationFactor:2    Configs:
Topic: test-kafka    Partition: 0    Leader: 2    Replicas: 2,3    Isr: 2,3
Topic: test-kafka    Partition: 1    Leader: 3    Replicas: 3,1    Isr: 3,1
[hadoop@server3 kafka]$

    可以看到test-kafka 有两个partition， partition 0 上 存在Leader 2 (Broker2)，因为副本数为2，所以Broker3也存了一份，
    这个Leader表示，我在Broker2这个节点找不到就到Broker3那里去找，提高了可靠性，不会因为某台机子宕掉了而停止工作

  6. 启动消费者与生产者

    server2为生产者，server3消费者–zookeeper 后面跟zk集群中某一台机子就行

    [hadoop@server2 kafka]$ bin/kafka-console-producer.sh --broker-list server3:9092 --topic test-kafka 
test kafka 
hello  I'm guin_guo
[hadoop@server3 kafka]$ bin/kafka-console-consumer.sh --zookeeper server1:2181 --topic test-kafka
test kafka 
hello  I'm guin_guo

在server2上输入(生产)内容，server3就显示(消费)出来了
