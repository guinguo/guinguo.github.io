Linux系统配置ssh免密码登录

发表于 2016-05-14   |   分类于 Linux   |  

一开始以为很简单，配置一下就好，没想到，一粗心，弄了有点久

　　首先呢，什么是SSH呢？–>传送门。然后，如果你每次登录远程服务器，都需要按(复杂的)密码，会很麻烦的，为了安全性，不会设置简单密码，可是每次都输入，很痛苦的。。。

好了，如果你了解了ssh之后，那么，如何生成密匙对呢？

利用git的ssh密匙对

　　打开PUTTYGEN，在Putty安装目录下，如果没有自行下载。然后点击顶部Conversions->Import Key,选择id_rsa，一般存在C盘的用户目录下的.ssh文件夹里面，点击Save private key
保存下来，用于authenticate的公匙。把id_rsa.pub里面那一行内容拷贝下来。跳过第2步。

自己生成密匙对

　　同样打开PUTTYGEN,点击Generate，注意，鼠标要在中间（进度条下面不断移动！），如果不这样做要等很久，上次有同学后台运行，等了快半个钟都没好，不知道老外怎么设计的这个软件。。。
,好了之后，点击Save private key,保存下来，用于authenticate的公匙。顺便吧key框里面的内容拷贝下来，保存也行，点击Save public key。然后continue~

配置服务器

好了，完成以上两步中的一步就可以继续了，打开，putty，用密码登录远程服务器操作。

建立~/.ssh目录（如果没有）

ls ~/.ssh
authorized_keys
#没有就
mkdir ~/.ssh
vim ~/.ssh/authorized_keys
#把刚才复制的内容，贴进去
:wq#保存退出

注意，这里有一个坑，就是鼠标右键粘贴时，前面几个字母可能会丢失，我也不知道为什么，一不注意，就像我一样GG了–>server refused our key，我跟这位up主一样去找各种原因，，，

重启sshd服务

service sshd restart

测试ssh公匙登录

　　打开Putty

然后就可以在putty中加入它了。然后点击open。

Using username "root".
Authenticating with public key "imported-openssh-key"
Last login: Fri May 13 22:29:27 2016 from xx.xx.xx.xx

中间如果出现什么问题可以参考 centos7 服务器部署ssh证书授权登录 进行更详细的配置。
